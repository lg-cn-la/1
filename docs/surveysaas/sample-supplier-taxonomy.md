# Sample Supplier / 样本供应方分类补充说明

> 本文档是 `product-positioning-summary.md` 的补充，用于解释为什么 Sampora 的目标客户不能只写 Panel Providers，也需要覆盖 Sample Suppliers。  
> 后续官网、Sales PDF、Demo 讲解脚本、品牌定位文档和 Agent 交接中，涉及目标客户、供应链位置和行业角色定义时，可引用本文档。

---

## 1. 结论

在 Sampora 的语境中，**Panel Provider** 和 **Sample Supplier** 不是完全同义词。

- **Panel Provider** 更强调拥有并运营自有 Panel / respondent asset 的团队。
- **Sample Supplier** 是更大的样本供给方概念，既可以包含 Panel Provider，也可以包含聚合商、调研分包执行团队、调查站 / 执行单元、API 对接供应商等。

因此，Sampora 对外目标客户建议使用：

**Panel Providers & Sample Suppliers**

不建议只写：

**Panel Providers**

因为这会漏掉通过外部供应商、渠道、API、调查站或合作资源进行样本交付的团队。

---

## 2. 推荐分类树

```text
Sample Supplier / 样本供应方
├─ Panel Provider / 自有 Panel 供应商
├─ Sample Aggregator / 样本聚合商
├─ Survey Station / 调查站或执行单元
├─ Research Subcontractor / 调研分包执行团队
└─ API-connected Supplier / API 对接供应商
```

说明：`Audience Source / Audience Owner` 可以作为更广义在线样本生态里的潜在供给来源，但不应作为 Sampora 当前官网首屏或核心目标客户分类。Sampora 当前更应该优先表达已经参与在线样本项目交付、供应商协作、执行追踪、审核、结算或 API 对接的团队。

---

## 3. 各角色解释

### 3.1 Panel Provider / 自有 Panel 供应商

拥有并运营自己的受访者池，通常管理 panelist / respondent 的注册、画像、标签、分层、活跃度、调查记录、奖励和提现。

在 Sampora 中对应能力包括：

- Panel / 会员管理
- 会员组、等级、标签或画像
- 会员调查记录
- 会员钱包、提现、账单
- 自有门户或调查站能力
- 将自有 Panel 用于项目交付

---

### 3.2 Sample Aggregator / 样本聚合商

不一定拥有完整自有 Panel，而是整合多个样本来源、供应商、渠道或 API，以完成客户项目的样本交付。

在 Sampora 中对应能力包括：

- 多供应商管理
- 项目调度与供应商分配
- 屏蔽不适合供应商
- 按供应商设置项目价格
- API 接收 / 分发项目
- 供应商结算、账单和财务记录

---

### 3.3 Survey Station / 调查站或执行单元

偏底层执行团队，可能负责会员招募、会员运营、项目执行、答题入口、会员激励和提现等。

在 Sampora 中对应能力包括：

- 会员 / respondent 管理
- 调查列表和参与记录
- 会员钱包、账单、提现
- 项目执行记录
- 员工或执行团队管理

---

### 3.4 Research Subcontractor / 调研分包执行团队

从甲方、研究公司或上游调研服务商承接项目，再分包或分发给下游样本供应商、调查站、Panel 或 API 供应商执行。

在 Sampora 中对应能力包括：

- 客户管理
- 供应商管理
- 项目接收与创建
- 项目分配、控价、屏蔽
- 项目执行追踪
- 审核、结算、账单、发票
- 通过 Partner Network 寻找供应商

---

### 3.5 API-connected Supplier / API 对接供应商

供应商不一定使用 Sampora，但如果已经与客户系统或 Sampora 通过 API 对接，项目被分配后可以通过 API 接收项目，并在自己的系统里看到或处理项目。

如果供应商也使用 Sampora，则可以在自己的 Sampora 工作台中直接接收和管理被分配的项目。

在 Sampora 中对应能力包括：

- API 项目分发
- 项目接收 / Project Intake
- 回调记录
- API 供应商交付追踪
- 后续结算与账单管理

---

## 4. 为什么 Sample Suppliers 是 Sampora 的目标客户

Sampora 的能力不只覆盖自有 Panel 的管理，也覆盖在线样本交付中的供应商协作、项目分配、API 分发、供应商控价、客户 / 供应商结算和合作资源网络。

已确认的关键产品能力包括：

- 项目管理
- 项目接收与 API 分发
- 供应商分配
- 屏蔽供应商
- 供应商级项目价格设置
- 供应商管理
- 客户管理
- Panel / 会员管理
- 项目结算、账单、钱包和财务日志
- Partner Network：帮助客户找供应商，帮助供应商找客户
- 外部云供应商 / 云客户建立合作后转为内部供应商 / 客户，并进入后续项目分配、控价、结算和账单流程

因此，Sampora 的目标客户不能只限定为 Panel Providers。否则会遗漏没有强自有 Panel、但通过渠道、外部供应商、调查站、API 或合作资源完成样本交付的 Sample Suppliers。

## 5. 对官网和品牌文案的影响

### 推荐主定位

**Online Sample Operations Platform for Panel Providers & Sample Suppliers**

### 推荐中文解释

面向自有 Panel 运营方与在线样本供应商的样本运营平台。

### 推荐二级说明

**Built for panel teams, sample suppliers, research subcontractors, and multi-source delivery teams managing online sample projects across clients, suppliers, panels, finance, and partner networks.**

### 不建议只写

**For Panel Providers**

原因：过窄，无法完整覆盖 Supplier Network 与 Enterprise 版本。

### 不建议在首屏优先写

**Audience Source / Audience Owner**

原因：它更像广义样本生态中的潜在供给来源，不一定是已经具备在线样本项目交付、审核、结算、API、供应商协作能力的当前核心目标客户。首屏更建议使用 Survey Station / Execution Unit 来覆盖站点、调查站和执行单元用户。

### 不建议优先写

**For Fieldwork Agencies**

原因：容易让海外客户联想到线下 fieldwork、CATI、CAPI 或访问员执行管理，而 Sampora 的核心边界是 online sample operations。

---

## 6. 与版本拆分的关系

### Panel 型版本

主要面向 Panel Provider / 自有受访者资产运营方，也可覆盖 Survey Station / 执行单元。

### Supplier Network 型版本

主要面向 Sample Supplier、Sample Aggregator、Research Subcontractor、API-connected Supplier 这类需要管理外部供应链和样本交付的团队。

### Enterprise / 完整型版本

主要面向同时拥有自有 Panel，又需要管理外部供应商、客户、API、财务、权限和 Partner Network 的综合型样本业务团队。

## 7. 首页拓扑图建议

首屏拓扑图中的 `SUPPLIER POOL` 建议改为：

**SAMPLE SUPPLY POOL**

节点建议使用：

```text
PANEL_01
owned panel

AGG_02
sample aggregator

STN_03
survey station

SUB_04
subcontractor

API_05
API supplier
```

这组节点比两个 Panel 和两个 Aggregator 更能体现 Sampora 对多类型在线样本供给来源的覆盖，也比 Audience Source 更贴近当前目标客户。

---

## 8. 最终判断

**Panel Provider 是 Sample Supplier 生态中的一种，但 Sample Supplier 不一定是 Panel Provider。**

因此，对外品牌定位采用 **Panel Providers & Sample Suppliers** 更准确，也更能覆盖 Sampora 的真实产品能力和版本结构。
